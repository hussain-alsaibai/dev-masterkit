## 💡 Lessons Learned (added to zero-dep-pattern + repo-creator skills)

### 1. Prometheus bucket formatting requires integer buckets as integers, not `1.0`
- Initial `_format_bucket(1.0)` produced `"1.0"`, but Prometheus convention is `"1"`.
- Trailing `.0` on integer buckets breaks Prometheus scrapers.
- **Fix:** in `_format_bucket`, render `int(b)` when `b == int(b) and abs(b) < 1e15`.
- **Lesson:** when matching existing protocols, study the spec for edge cases (integer-valued floats here).

### 2. `from concurrent.futures import TimeoutError as CFTimeoutError` shadows builtin
- I imported `concurrent.futures.TimeoutError` and renamed it `CFTimeoutError`, but the `TimeoutError` builtin (== `OSError` subclass) is different.
- **Lesson:** when aliasing stdlib names, prefer explicit suffixes (`CF`, `STD`) over single-letter prefixes.

### 3. Test isolation requires per-test registry, not module-level
- First pass used `REGISTRY` (module global). When two tests created `Counter("x", ...)` they collided.
- **Fix:** every test gets `fresh_registry()` returning a new `MetricRegistry()`.
- **Lesson:** any singleton that accumulates state needs an isolation pattern in tests. Either per-test reset, per-test fixture, or per-test new instance. Pick one.

### 4. `Future.result(timeout=…)` does NOT cancel the worker thread
- The timeout cuts off the *wait* but the worker keeps running.
- The leaked worker is a daemon in the shared `ThreadPoolExecutor`, so it dies with the process.
- **Lesson:** document the "worker leak" honestly in the README. Don't pretend the thread is killed when it isn't.

### 5. AST-based zero-deps assertion is more reliable than grep
- A simple grep for `^import`/`^from` misses indented imports inside `try/except` blocks, function bodies, etc.
- AST walks the whole tree and ignores comments/strings.
- **Pattern:** `ast.walk(tree)` over `Import`/`ImportFrom`, allow only stdlib + `__future__`.

### 6. `with assertRaises(...)` wraps the entire block — including `with cm:`
- Test `with self.assertRaises(TimeoutExceeded): with soft_timeout(...): d.check()` works because `assertRaises` catches what propagates out of the `with cm:` block.
- **Lesson:** double-`with` is fine; just make sure the inner one raises.
