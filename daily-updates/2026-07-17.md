## Why It Matters

The tiny-* ecosystem is strongest when it gives autonomous agents small,
auditable building blocks:

- `tiny-router` owns the HTTP gateway.
- `tiny-validator` owns payload contracts.
- `fast-cache` owns bounded reuse and dedupe.
- `tiny-log`, `tiny-trace`, `tiny-otel`, and `tiny-budget` make decisions
  observable and bounded.

Together, they map well to the developer-tool demand forming around agent
runtime governance without requiring a platform rewrite.

## New reusable artifact

### `prompts/openclaw-self-version-check.md`

Captures the correct way to detect OpenClaw upstream updates without
trusting stale build-time paths. Today's cron task still referenced
`/skeleton/.npm-global/lib/node_modules/openclaw/package.json`, which
no longer matches the runtime install at `/usr/local/bin/openclaw`
→ `/app/openclaw.mjs`. The prompt ships a 3-line recipe
(`openclaw --version` + GitHub Releases API + comparison) plus the
verification steps.

**Why it's worth a prompt, not just a daily note:** every OpenClaw
container hits this same path drift; the recipe is short enough to
inline anywhere; and getting it wrong silently skips update alerts.
