### New repos (live on GitHub)

#### tiny-budget — `github.com/hussain-alsaibai/tiny-budget`
Zero-dependency runtime budget enforcement for AI agents. Tracks USD +
token spend against per-task limits, throws `BudgetExceeded` on overflow,
persists state atomically so a crashed agent doesn't double-spend on restart.

- USD + token limits with `BudgetExceeded` exception
- Atomic JSON persistence (write-temp + rename) for crash safety
- Per-task isolation (different `taskId` = fresh budget)
- Built-in pricing for OpenAI/Anthropic/Google; `setPricing()` for the rest
- `chargeUsage()` accepts both OpenAI and Anthropic usage shapes
- `chargeFlat()` for opaque-cost APIs
- Warning callback fires once per threshold crossing (no spam)
- CLI: `tiny-budget simulate --limit-usd=1.00 < charges.jsonl` + `tiny-budget price`
- 16 tests passing

The thing this prevents is the "$400 LLM bill from a runaway agent loop"
failure mode. For cron-driven agents this is genuinely a missing primitive.

#### tiny-eventbus — `github.com/hussain-alsaibai/tiny-eventbus`
Zero-dependency durable pub/sub with replay. JSONL log + topic filtering +
automatic rotation. Designed for agent workflows where multiple crons share
state and a restart shouldn't lose pending work.

- Append-only JSONL log for crash-safe event history
- Topic-based pub/sub with wildcard (`subscribeAll`)
- Replay on subscribe (configurable `fromOffset` for cursor-based consumers)
- `once()` with predicate + timeout for awaitable one-shots
- `history()` with topic / topicPrefix / limit filters
- Automatic log rotation at `maxLogBytes`
- Subscriber errors don't kill dispatch (emitted as `subscriber_error`)
- CLI: `tail` for live + `history --topic=foo` for queries
- 17 tests passing

## Trend Call

The agent ecosystem is converging on three primitives that are missing from
the stdlib and from the big "framework" packages (LangChain, LlamaIndex,
Mastra, etc.):

1. **Cost enforcement at the call site.** Every serious agent runtime ships
   some version of this now (LiteLLM's budget callbacks, LangSmith's
   rate limits, Helicone's budgets). The standalone "put a Budget object
   around your LLM call" pattern is the right shape, and there was no
   zero-dep implementation.
2. **Durable intra-process pub/sub.** In-process `EventEmitter` works for
   one run; Redis Streams works for distributed systems. The middle
   "JSONL log + replay" tier is exactly what agent workflows need and
   what Kafka/Redis overkill for.
3. **Embedded K/V with replay safety.** SQLite is the answer at scale; the
   400-line WAL+snapshot K/V is the answer when you want to audit the
   whole storage layer in 5 minutes.

Patterns I'm seeing in 2026 H2:

- **Agents vendoring 5-15 single-file libs** instead of pulling in 1 framework
- **Bounty/security work** driving demand for self-contained repro harnesses
  (single-file CLIs + tiny-routers + validators)
- **Cron-driven workflows** that need: idempotency, timeouts, cost caps,
  structured event logs, and durable state — all of which now exist in
  the ecosystem
