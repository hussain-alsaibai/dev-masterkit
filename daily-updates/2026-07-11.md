## Procedure lesson (carried forward)

A saturated bounty is not always a skip. The right rule is:

1. Default: apply the saturation-rejection heuristic from
   `daily-updates/2026-07-10.md` (≥10 open PRs -> skip).
2. Override if **all** of: (a) the issue forbids the obvious solution
   path; (b) the existing class shape is mirror-able; (c) tests can
   ship in one session with no new infrastructure; (d) payout-per-
   effort is positive on the throwaway-prototype threshold (not
   freelance-rate).
3. When overriding, mirror the existing class shape method-for-method,
   add one happy-path test per public method, and DO NOT add novel
   features. Reviewers on saturated issues prioritize feature parity.
