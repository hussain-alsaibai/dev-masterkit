## Lessons Learned

### YAML Subset Parser Tuple Bug
- **Problem:** `_yaml_parse_block` returns `(value, position)` but every caller
  had `if isinstance(parsed, dict): ... else: raise`. Since
  `isinstance(tuple, dict)` is always False, the parser always raised
  `ValueError: expected dict, got tuple`.
- **Fix:** Unpack the tuple at every call site:
  `parsed, _ = _yaml_parse_block(...)`
- **Why not patch the parser?** The tuple contract is correct — it conveys
  both the value and the position for nested parsing. Callers were wrong.
- **Last verified:** 2026-06-30

### Boolean CLI Flag Trap
- **Problem:** argparse expects a value when an option is registered with
  `type=bool`. So `--shout Alice` errors with "expected one argument"
  because argparse tries to parse "Alice" as the bool.
- **Fix:** In the CLI wrapper, detect `m["type"] is bool` and use
  `action="store_true"` instead of `type=bool`.
- **Applies to:** any Click-style decorator wrapper around argparse.
- **Last verified:** 2026-06-30

### TTY Detection in Test Runners
- **Problem:** `sys.stdout.isatty()` returns False under pytest, IDE test
  runners, and cron. So `app.color("hi", "green")` silently emits no ANSI codes
  during tests, making color regressions invisible.
- **Fix:** Provide a force override env var (`TINY_CLI_FORCE_COLOR=1`) AND
  honor `NO_COLOR` (https://no-color.org).
- **Precedence:** force ON → `NO_COLOR` → TTY → default OFF
- **Last verified:** 2026-06-30
