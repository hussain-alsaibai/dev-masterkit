## New Repositories

### 1. tiny-worker
- **URL**: https://github.com/hussain-alsaibai/tiny-worker
- **Description**: Zero-dependency background task queue for Python — like Celery, but in one file
- **Size**: 9.4KB
- **Key Features**:
  - Thread pool with configurable workers
  - Retry with exponential backoff
  - Token bucket rate limiting
  - Per-task timeouts
  - Completion/error callbacks
  - Batch operations (`.map()`)
  - Context manager support
- **Use Case**: Background job processing without Redis/Celery infrastructure

### 2. tiny-events
- **URL**: https://github.com/hussain-alsaibai/tiny-events
- **Description**: Zero-dependency event bus for Python — like Redis Pub/Sub, but in one file
- **Size**: 7.9KB
- **Key Features**:
  - Pub/sub with pattern matching
  - Wildcards (`user.*`, `*.created`, `*`)
  - Priority levels (LOW, NORMAL, HIGH, CRITICAL)
  - One-shot subscriptions
  - Blocking `wait_for()` with timeout
  - Event history for replay/debug
  - Thread-safe
- **Use Case**: Decoupled event-driven architecture without message brokers

### 3. tiny-http
- **URL**: https://github.com/hussain-alsaibai/tiny-http
- **Description**: Zero-dependency HTTP client for Python — like requests, but in one file
- **Size**: 9.9KB
- **Key Features**:
  - GET, POST, PUT, DELETE, PATCH
  - JSON request/response
  - Automatic retries with exponential backoff
  - Token bucket rate limiting
  - Session cookies and headers
  - Query parameter encoding
  - Timeout handling
  - Request statistics
- **Use Case**: HTTP requests without external dependencies (requests, urllib3)
