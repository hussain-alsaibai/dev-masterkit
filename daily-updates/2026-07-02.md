## 💡 Lessons Learned (added to repo-creator + zero-dep-pattern skills)

### 1. `composed(a, b, c)` applies factories RIGHT-TO-LEFT
- Matches stacked `@a @b @c def f:` semantics
- **Implementation:** iterate `reversed(factories)`; display `reversed(applied)` for outer-first peek
- **Gotcha:** `deco.__name__ = factory.__name__` is required (the inner `def deco(fn)` shadows otherwise)
- Last verified: 2026-07-02

### 2. `functools.partial` of a factory ≠ a wrapped decorator
- `partial(add_n_factory, 3)` returns the inner `deco` closure, not a wrapper
- Real use: `partial(some_decorator, n=3)` where the decorator signature is `(fn, n=...)`
- Last verified: 2026-07-02

### 3. AWS SigV4 by hand is ~50 LOC and one action
- 4-step HMAC chain: `k_date` → `k_region` → `k_service` → `k_signing`
- Canonical request → string to sign → signature
- Stdlib `hmac` + `hashlib` is enough
- **Do not** extend it to a general AWS SDK — surface area is too large
- Last verified: 2026-07-02

### 4. W3C `traceparent` version field must be `"00"`
- Anything else → ignore the parent (return `INVALID_SPAN_CONTEXT`)
- The first regex required 4 components but didn't validate `version == "00"`
- Result: invalid parents silently got used
- Last verified: 2026-07-02

### 5. `[[` is bash-only — use `case` or Python in `sh` scripts
- `set -e` and `[[` are bash builtins
- Default `sh` in the container is `dash`, not bash
- Use `case` statements or rewrite the loop in Python
- Last verified: 2026-07-02

### 6. Cross-link automation needs format detection
- 12 existing repos had 3 different "Ecosystem" section formats
- Format 1: `| [name](url) | desc |` (most repos)
- Format 2: `- [**name**](url) — desc` (tiny-cli etc., bold bullet)
- Format 3: `- [name](url) — desc` (tiny-agent, plain bullet)
- tiny-embed lists other repos but NOT itself (no self-reference)
- Solution: `crosslink.py` detects format from existing lines and re-uses it
- Last verified: 2026-07-02
