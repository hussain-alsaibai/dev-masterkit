### MEMORY (50K rows):
| Engine | Memory | vs DuckDB |
|--------|--------|-----------|
| SnapDB Row | 0.6 MB | **17x more efficient** |
| SnapDB Col | 1.5 MB | **7x more efficient** |
| SQLite | 9.4 MB | 1.1x |
| DuckDB | 10.7 MB | baseline |
| Pure Dict | 16.8 MB | 0.6x |

### Key Insight
SnapDB's killer advantage is **memory efficiency** — 17x less than DuckDB, 28x less than pure dict. DuckDB dominates raw analytical throughput (96M rows/sec) but is 17x heavier on memory. SnapDB fills the niche of ultra-low-memory embedded storage in pure Python.
