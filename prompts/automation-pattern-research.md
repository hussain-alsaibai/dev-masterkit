# automation-pattern-research — Prompt for Systematic AI Automation Research

## Description

Guides systematic research into AI agent automation patterns, use cases, and community practices. Produces structured reports with actionable findings.

## The Prompt

```
Research AI automation patterns for: {domain}

Deliverables:
1. **Use Case Inventory** — List 5-10 specific automations people are actually running
2. **Tool Stack** — What tools/skills are commonly used for each use case
3. **Impact Metrics** — Any reported time savings, error reductions, or efficiency gains
4. **Implementation Difficulty** — Easy / Medium / Hard per use case
5. **Our Applicability** — Which ones match our current setup and goals

Research sources:
- GitHub repos (awesome-* lists, example projects)
- Reddit discussions (r/OpenClaw, r/LocalLLaMA, r/MachineLearning)
- Blog posts and case studies
- Official documentation and community forums

Report format:
| Use Case | Tools | Impact | Difficulty | Applicable |
|----------|-------|--------|------------|------------|

Include:
- Links to source material
- Community quotes if available
- Specific commands or configurations used
- Pitfalls or limitations mentioned

Rules:
- Only document patterns with evidence (not speculation)
- Note the date of each source
- Flag anything that seems outdated or deprecated
- Suggest 2-3 next steps based on findings
```

## When to Use

- Exploring new automation domains
- Benchmarking our setup against community practices
- Justifying time investment in a new automation
- Building skills library based on proven patterns
